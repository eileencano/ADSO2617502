INICIANDO CON GIT

Git s un control de versionesque permite a equipos desarrolladores trabajar de manera remota en un mismo proyecto.permite hacer contribuciones de codigo de manera local y luego replicarlos a un repositorio remoto

PRICIPALES COMANDOS CON GIT

  • git init:fuciona para inicializar un proyecto en git.
  • git config --global user.email"email":para identificar y configurar globalment el email con el repositorio.
  • git config --global user.name"email":para identificar y configurar globalmente el nombre de usuario con el repositorio local.
  • git config --global user.email:verifica el email agregado.
  • git config --global user.name:verifica el nombre de usuario agregado.
  • git add .:fuciona para agregar todos los archivos modificados y nuevos en el repiositorio local de git.
  • git status:permite verificar el estado del repositorio.
  • git commit -m"comentarios":permite guardar los cambios realzados en el repositorio local con un mesaje resumido.
  • git log":muestra el historial de cambios en el repositorio local.
  • git branch": muestra la rama del repositorio local.
  • git branch checkout -b nombre_rama":cambiar el nombre de la rama de un repositorio local
  • git checkout -- .":deshacer los cambios en un archivo no confirmado
  • git checkout remote add origin url": el repositorio local se agrega a un repsitorio remoto como GITHUB.
  • git push -u origin main:
    • enviar los commit del repositorio local al repositorio remoto de GITHUB
